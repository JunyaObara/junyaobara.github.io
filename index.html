Held-out Cold Test for a Statute-Grounded Privacy Risk Triage PoC

This PoC converts early-stage software requirements into structured privacy risk triage for BIPA, CIPA, and VPPA. Its value is not a vanity accuracy number; it is an artifact-backed, traceable evaluation pipeline with explicit benchmark gates that prevent unsupported claims.

What the system does

  • Reads requirement/spec fragments and performs first-pass privacy issue spotting.
  • Produces one external label: BLOCKER / NEEDS_REVIEW / CLEARED.
  • Routes review cases to DEV / LEGAL / POLICY when needed.
  • Returns statute-grounded basis citations plus remediation-oriented recommendations.
The objective is faster, more traceable triage — not autonomous legal advice.

What this result means — and what it does not mean

What this result supports

  • A real held-out evaluation was executed under cold conditions.
  • The evaluation pipeline is traceable: protocol, cache state, checklist source, and leakage checks are recorded.
  • Basis-citation quality is strong enough to demonstrate statute-grounded reasoning value in a PoC setting.
  • The system distinguishes run success from benchmark readiness instead of hiding unresolved risk.

What this result does not support

  • It does not prove production-ready legal accuracy.
  • It does not prove broad real-world performance on lawyer-authored free-form documents.
  • It does not yet provide all-law support/explainability scoring.
  • It does not yet prove full KB control-plane provenance.
Current reasons for claim_ready = false: coverage warnings, incomplete KB provenance, 55 conflicting answer buckets in the validation-derived checklist source, and a template-only benchmark family.

Protocol

Held-out test, evaluation_is_in_sample=false, workers=1, no fast mode, cold primary run.

Inputs

Frozen validation checklists, hardened cache namespace, stripped runner input, no bootstrap overlap with the test split.
